REINFORCEMENT LEARNING · CLAUDE CODE CRAWLER

Learning to play a roguelike — and matching the search oracle

A learned, net-guided search agent that matches pure MCTS on the live game: 100% at base (with fewer simulations), ~80% at 1.5× difficulty, ~45% at 2.0×. Built and validated entirely in Node — and the real story is how many "ceilings" turned out to be measurement artifacts.

The challenge

The engine is a pure, deterministic reducer; all randomness lives in seeded RNG streams. We feed the agent the full game state except the RNG — so the future is hidden. That makes this a long-horizon planning problem: ~7–8 map nodes, hundreds of decisions, and a single win/loss reward at the very end. The decisive choices (which path, which card, when to rest) pay off far later — so lookahead matters, which is exactly why search dominates a reactive policy and why the network's real job is to guide search.

What we built

Encoder + action mask
State → fixed vector (vocab from the content registry; RNG excluded). 73-slot flat action space over the engine's legal actions, with an append-only vocab manifest + fingerprint guard.
Networks
An MLP and an attention-pooling net over learned entity embeddings — both with hand-derived backprop validated by a finite-difference gradient check.
Search family
Clairvoyant MCTS, net-guided PUCT, hybrid (net priors + rollout value), ISMCTS (determinized, fog-of-war), and net-guided determinized search for self-play.
Training
Behavioral cloning, distillation, determinized-Q targets, DAgger (on-policy), and full AlphaZero self-play — plus a Python actor/learner scaffold.

The journey — hypotheses & verdicts

Each step tested a claim. Most were refuted by a controlled experiment — which is the point.

  1. Pure MCTS solves base difficulty. confirmed (100%)
  2. A learned value head can replace rollouts. partly — capped ~85%; value-at-leaf is the limiter
  3. Positional-hand encoding helps the policy. refuted — noise
  4. A "~30% no-search ceiling" is fundamental. refuted (unlearnable targets + distribution shift + a silent poison/dexterity encoding bug + too-few seeds)
  5. Clone a deterministic, learnable expert (greedy) with DAgger. 70% — beats the heuristic
  6. Hybrid PUCT (priors + rollout value). 100% at base @160 sims
  7. Hard difficulty needs a learnable hard expert → ISMCTS. priors lift hybrid 63%→83%
  8. One unified net across difficulties. 100% base / ~80% hard
  9. Full AlphaZero self-play (determinized → learnable). self-improves, no collapse
  10. "We don't match MCTS at hard." refuted — measurement error (two different difficulty mechanisms compared)

Why it works

The net guides search; it doesn't play alone

A single forward pass can't do lookahead, so a no-search policy is decent at base (70%) but weak at hard. Its real value is as a prior: hybrid PUCT uses the net's priors to focus the search and honest rollouts for leaf values — best of both, and more sim-efficient than blind MCTS.

Targets must be learnable

Cloning clairvoyant MCTS fails: its chosen action depends on the hidden future, so the target isn't a function of what the net sees. Determinization — re-sampling the hidden future each search iteration (ISMCTS / determinized-Q) — makes targets an expectation over futures, i.e. a function of the observable state, hence learnable.

Train on your own states (DAgger)

Offline imitation cascades: small errors take the agent to states the dataset never covered. DAgger labels the states the agent actually visits, eliminating the cascade.

Self-play that doesn't collapse

AlphaZero self-play with net-guided determinized search produces learnable, on-policy targets — so, warm-started, it self-improves with no value collapse and no distribution cascade (the failure modes that broke every earlier attempt).

Methodology lessons

Measure honestly. Fix one difficulty mechanism + seed set + sim budget before quoting a "ceiling." Three confident ceilings here were artifacts of inconsistent setups or too-few seeds.

Learnable targets, on-policy data. Determinize unlearnable experts; train on the agent's own distribution.

Expert consistency matters for cloning — a deterministic heuristic clones far better than a strong-but-stochastic search.

Audit encoder↔game alignment on every update — a hardcoded status list silently dropped real mechanics; a structural fingerprint now guards it. And the model is tiny (~10k–300k params): GPU is a convenience, not a requirement.
